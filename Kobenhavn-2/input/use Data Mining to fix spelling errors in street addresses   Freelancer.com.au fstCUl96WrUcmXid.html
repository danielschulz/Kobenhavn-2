Contest Brief

We have a system that uses Machine Learning (WEKA) to cleanse missing and incorrect data in an analytical database.

The goal of this project is to improve and expand the Java program which cleans up Street Address, Latitude, and Longitude to create a single canonical representation of physical locations (so same location will always have same spelling and same Lat/Lng regardless of minor data errors). Goal of cleansing program is to select canonical spelling and Lat/Lon for each address and to detect and fix outliers. Please note "CANONICAL" spelling means same spelling used for all addresses that refer to the same physical location. Actually the most important feature is the Spelling of street address. The LAT/LNG is nice too but not as critical, and could be a tie-breaker.

This solution is for a commercial application. Using external services like Google Maps in the solution is not permitted. The current version here already outperforms Google Maps in accuracy for correction of street names (for example, "South C Ave" is incorrectly classified by Google as "S AVE E", when correct answer is "S AVE C"). So please do not insist about using Google Maps or other services, they are inadequate.

You may use external services to compare your results, provided that they are not made part of the solution.

Contest Entries will be judged on five measures: 1) Correctness of results, 2) Elegance of Solution and Code, 3) Memory and Run-Time Requirements, 4) Originality (in case of similar entries by different contestants, the earliest entry will be chosen), 5) Presentation of results (easy to understand).

ATTACHMENTS: Java program and Database dump, comments from previous contest, and sample program output. Two datasets - partial and full.

The final solution does not have to be in Java/WEKA. You may remove WEKA and use a different open-source library if you wish. Or use something like R, or RapidMiner, or Apache Hadoop. Or C++. I'm willing to accept a low-cost commercial solution (Microsoft SQL Server DMX) if it is superior to the open-source solutions.

HINT: I believe solutions should look at cleansing low-cardinality items first, like the Zip Code and CP, then re-using cleansed value as input to street name cleanser. Also different algorithms may have better or worse performance. Also look at "Ensemble" algorithms!

Big Data, Data Mining, Java, Machine Learning, MySQL
